# omni-runpod-worker

> [OmniVoice](https://huggingface.co/k2-fsa/OmniVoice) TTS running on [ComfyUI](https://github.com/comfyanonymous/ComfyUI) as a serverless API on [RunPod](https://www.runpod.io/)

Zero-shot multilingual text-to-speech with voice cloning and voice design, supporting 600+ languages. Built on the [ComfyUI-OmniVoice-TTS](https://github.com/Saganaki22/ComfyUI-OmniVoice-TTS) custom node.

## Features

- **600+ Languages** — Broadest language coverage among zero-shot TTS models
- **Voice Cloning** — Clone any voice from 3-15 seconds of reference audio
- **Voice Design** — Create synthetic voices from text descriptions
- **Multi-Speaker Dialogue** — Generate conversations between multiple speakers
- **Fast Inference** — RTF as low as 0.025 (40x faster than real-time)
- **Audio + Image Output** — Handler supports both audio and image outputs from ComfyUI workflows

## Quickstart

1. Build the Docker image on a Hetzner server (see [Building the Docker Image](#building-the-docker-image))
2. Push the image to Docker Hub
3. Create a RunPod serverless template with the Docker image
4. Create a RunPod endpoint from the template (recommended: GPU with 8+ GB VRAM)
5. Send workflow requests to your endpoint

## Building the Docker Image

### On a Hetzner Server (Recommended)

SSH into your Hetzner server and run:

```bash
export DOCKERHUB_USERNAME="your-dockerhub-user"
export DOCKERHUB_TOKEN="your-dockerhub-access-token"
export IMAGE_TAG="your-user/omni-runpod-worker:latest-omnivoice-bf16"

curl -fsSL https://raw.githubusercontent.com/Jmendapara/omni-runpod-worker/main/scripts/build-on-pod.sh | bash
```

### Manual Build

```bash
git clone https://github.com/Jmendapara/omni-runpod-worker.git
cd omni-runpod-worker

# Build with OmniVoice-bf16 (default, ~2GB model, recommended)
docker buildx build --target final --build-arg MODEL_TYPE=omnivoice-bf16 -t omni-runpod-worker:bf16 .

# Or build with OmniVoice fp32 (~4GB model, higher quality)
docker buildx build --target final --build-arg MODEL_TYPE=omnivoice-fp32 -t omni-runpod-worker:fp32 .
```

### Model Variants

| MODEL_TYPE | Model Size | Whisper | Total Image Size | Description |
|---|---|---|---|---|
| `omnivoice-bf16` | ~2 GB | ~1.5 GB | ~12 GB | BFloat16 quantized (recommended) |
| `omnivoice-fp32` | ~4 GB | ~1.5 GB | ~14 GB | Full precision |

Both variants include Whisper large-v3-turbo for automatic transcription during voice cloning.

## API Specification

The worker exposes standard RunPod serverless endpoints (`/run`, `/runsync`, `/health`).

### Input

```json
{
  "input": {
    "workflow": {
      "1": {
        "inputs": {
          "model": "OmniVoice-bf16",
          "text": "Hello! This is OmniVoice speaking.",
          "steps": 32,
          "guidance_scale": 2.0,
          "speed": 1.0,
          "device": "auto",
          "dtype": "auto",
          "seed": 42,
          "keep_model_loaded": true
        },
        "class_type": "OmniVoiceLongformTTS",
        "_meta": { "title": "OmniVoice Longform TTS" }
      },
      "2": {
        "inputs": {
          "filename_prefix": "ComfyUI_OmniVoice",
          "audio": ["1", 0]
        },
        "class_type": "SaveAudio",
        "_meta": { "title": "Save Audio" }
      }
    }
  }
}
```

| Field | Type | Required | Description |
|---|---|---|---|
| `input.workflow` | Object | Yes | ComfyUI workflow exported via Workflow > Export (API) |
| `input.images` | Array | No | Input files (images or audio for voice cloning). Each object has `name` and `image` (base64) keys |
| `input.comfy_org_api_key` | String | No | Per-request Comfy.org API key |

### Output

```json
{
  "id": "sync-uuid-string",
  "status": "COMPLETED",
  "output": {
    "audio": [
      {
        "filename": "ComfyUI_OmniVoice_00001_.flac",
        "type": "base64",
        "data": "ZkxhQwAAACIQABAAAA..."
      }
    ]
  }
}
```

| Field | Type | Description |
|---|---|---|
| `output.audio` | Array | Audio files generated by the workflow (base64 or S3 URL) |
| `output.images` | Array | Image files, if workflow produces any |
| `output.errors` | Array | Non-fatal errors or warnings |

Each item in `output.audio` or `output.images`:

| Field | Type | Description |
|---|---|---|
| `filename` | String | Original filename from ComfyUI |
| `type` | String | `"base64"` or `"s3_url"` |
| `data` | String | Base64 encoded data or S3 URL |

## Usage

### Generate Speech (Sync)

```bash
curl -X POST \
  -H "Authorization: Bearer <api_key>" \
  -H "Content-Type: application/json" \
  -d @test_input.json \
  https://api.runpod.ai/v2/<endpoint_id>/runsync
```

### Voice Cloning

To clone a voice, include reference audio in the `images` array (the upload endpoint accepts any file type) and reference it in your workflow:

```json
{
  "input": {
    "images": [
      {
        "name": "reference_voice.wav",
        "image": "data:audio/wav;base64,UklGRi..."
      }
    ],
    "workflow": {
      "1": {
        "inputs": {
          "model": "OmniVoice-bf16",
          "text": "Hello in a cloned voice!",
          "ref_text": "",
          "steps": 32,
          "guidance_scale": 2.0,
          "speed": 1.0,
          "device": "auto",
          "dtype": "auto",
          "seed": 42,
          "keep_model_loaded": true
        },
        "class_type": "OmniVoiceCloneTTS",
        "_meta": { "title": "OmniVoice Voice Clone TTS" }
      },
      "2": {
        "inputs": {
          "image": "reference_voice.wav",
          "upload": "image"
        },
        "class_type": "LoadImage",
        "_meta": { "title": "Load Reference Audio" }
      },
      "3": {
        "inputs": {
          "filename_prefix": "ComfyUI_OmniVoice",
          "audio": ["1", 0]
        },
        "class_type": "SaveAudio",
        "_meta": { "title": "Save Audio" }
      }
    }
  }
}
```

## Getting the Workflow JSON

1. Open ComfyUI in your browser
2. Build your OmniVoice workflow using the available nodes
3. Select **Workflow > Export (API)**
4. Use the downloaded JSON as the `input.workflow` value

### Available OmniVoice Nodes

| Node | Description |
|---|---|
| OmniVoice Longform TTS | Long-form TTS with smart chunking |
| OmniVoice Voice Clone TTS | Clone voice from reference audio |
| OmniVoice Voice Design TTS | Design voices from text descriptions |
| OmniVoice Multi-Speaker TTS | Multi-speaker dialogue generation |
| OmniVoice Whisper Loader | Pre-load Whisper for auto-transcription |

## Configuration

| Environment Variable | Default | Description |
|---|---|---|
| `REFRESH_WORKER` | `false` | Clean state after each job |
| `COMFY_LOG_LEVEL` | `DEBUG` | ComfyUI log verbosity |
| `SERVE_API_LOCALLY` | `false` | Enable local API for development |
| `NETWORK_VOLUME_DEBUG` | `false` | Enable network volume diagnostics |
| `BUCKET_ENDPOINT_URL` | - | S3 endpoint for file upload |
| `BUCKET_ACCESS_KEY_ID` | - | S3 access key |
| `BUCKET_SECRET_ACCESS_KEY` | - | S3 secret key |
| `COMFY_ORG_API_KEY` | - | Default Comfy.org API key |
| `WEBSOCKET_RECONNECT_ATTEMPTS` | `5` | WebSocket reconnection attempts |
| `WEBSOCKET_RECONNECT_DELAY_S` | `3` | Delay between reconnection attempts |
| `COMFY_RESTART_DELAY` | `5` | Seconds before restarting crashed ComfyUI |
| `COMFY_MAX_RAPID_RESTARTS` | `5` | Max rapid restarts before giving up |
| `COMFY_STARTUP_CHECK_MAX_TRIES` | `120` | Seconds to wait for ComfyUI to start |

## Local Development

```bash
# Build the development image
docker build --target final --build-arg MODEL_TYPE=omnivoice-bf16 -t omni-runpod-worker:dev .

# Run with docker-compose
docker-compose up

# ComfyUI UI: http://localhost:8188
# RunPod API: http://localhost:8000
```
